Rules & FAQ

Policy, workshop, and clarifications

Fair, reproducible, and transparent evaluation guidelines for all participants.

Rules

Competition rules

Each team must register once and use exactly one team ID.
Each submitted archive must follow the task format and include the team ID.
Test ground truth is private.
Participants must not infer, reconstruct, scrape, or obtain test labels.
All training data, pretraining data, model checkpoints, and external resources must be declared.
External public data is allowed but flagged separately in leaderboard analysis.
Test-time adaptation is allowed only without test labels, manual annotation, or information from future leaderboard evaluations.
Manual per-sample editing of test predictions is prohibited.
Public leaderboard submissions will be rate limited.
Final submissions are frozen after the deadline.
Award candidates must have an accepted workshop paper and submit a complete reproducibility package (training and evaluation scripts, dataset declaration, resource requirements, and environment setup). See the reproducibility requirements for details.

Rule changes

Rule changes will be versioned, dated, and announced to registered teams.

Workshop

Workshop and review

  • Workshop papers are 4–6 pages.
  • Reviewing is single-blind.
  • Acceptance depends on technical soundness, clarity, reproducibility, and relevance to robust dense correspondence, not only leaderboard rank.
  • Accepted papers may be forwarded for the official PMLR volume of NeurIPS workshop proceedings.
FAQ

Frequently asked questions

Can I participate in only one task?

Yes. Single-task participation is allowed. A cross-task leaderboard will rank teams that submit to all three quantitative tasks.

Is external data allowed?

Yes, if it is public and declared. Entries using external data will be flagged separately.

Is test-time adaptation allowed?

Yes, but only without test labels, manual annotation, or future leaderboard information.

Do I need to submit a paper?

Yes. Each quantitative or exploration entry must submit a 4–6 page single-blind workshop paper.

What must the reproducibility package include?

With the camera-ready deadline, each entry must submit training and evaluation scripts, a complete dataset declaration, documented resource requirements, and a README with a pinned environment—either via pyproject.toml or explicit conda instructions with library versions. See the reproducibility requirements for the full checklist.

How are ties broken?

First by the normalized RobustSpring term, then by the normalized Spring term.

Where should questions be asked?

Through GitHub issues and public question threads, or the contact email roco-spring-org@googlegroups.com.
